MODULARIZED HTML COMPILER
Breve Explicación
En base a la necesidad de trabajar en un proyecto de sólo HTML 5 (html + css3 + js), y
manteniendo la idea de trabajar las vistas (HTML) de forma modularizada se requiere la
realización de un compilador de HTML5.
Descripcion
HTML5Compiler será una aplicación que compilara un proyecto HTML modularizado a una
aplicación de archivos estáticos de HTML. Ésta recibirá la ruta de un directorio, en donde se
encuentre la raíz de un proyecto HTML5, y tendrá el objetivo de construir una aplicación en base
a archivos de html modularizados y los tags que los relacionan.
Componentes
Se define uno o más tags al inicio de cada archivo que darán instrucciones al compilador del
comportamiento del archivo.
Estos tags tendrán la estructura de:
<# property=”” #>
Las posibles propiedades a ser incluidas en su versión beta serán:
1. isSection
Boolean que indica si el archivo en el que me encuentro genera una sección, osea si a partir de
este archivo voy a generar un html.
2. type
Este tag indica si el archivo al que se hace referencia será incluido en nuestro html parcial (a
partir de ahora modulo) o nuestro módulo será envuelto por el mismo. Estos dos valores toman
respectivamente los nombres de “Include” y “WrappedBy”
3. position
Este tag es dependiente de el tag type, en caso de que el tag type sea del tipo “WrappedBy”
este tag debería estar siempre presente, ya que indica en qué parte del módulo padre será
incluido, el valor será un identificador de la sección. En el módulo padre la definición de la
ubicación será de esta forma: <!--#SectionName#--> (se envuelve en un comentario de HTML
ya que la sección puede no siempre ser utilizada).
4. fileName
Ruta relativa, a la raíz del proyecto, del archivo que se desea incluir o sobre el cual se desea
envolver.
5. overwriteParent
Identificador que indica que indica que y cual tag del padre o cualquier nivel superior tiene como
objetivo sobrescribir.
6. tagName
Identificador que indica que indica que el tag es sobreescribible y ademas describe el nombre
que lo identifica.
Ejemplo de un tag común de una sección a partir de la cual se generará un HTML:
<# isSection=”true” #>
<# type=”WrappedBy” position=”center” fileName=”/modules/body/default.html”  #>
<# type=”WrappedBy” position=”body” fileName=”/modules/header/notdefault.html”
overwriteParent=”header”  #>
* <# isSection #> también es válido, el default es true en todo boolean.
El proyecto modularizado deberá tener uno y solo un index.html el cual se encontrará en la
carpeta raíz del proyecto. Este index deberá mantenerse lo más genérico posible y será la base
en donde deba finalizar la compilación modularizada del html.
